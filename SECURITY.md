# SafeStack Security Policy

## Scope

This repository contains public SafeStack materials, including:
- static web interface
- manifesto documents
- non-sensitive assets

No infrastructure, credentials, or operational systems are hosted here.

---

## Security Model

SafeStack follows a minimal attack surface principle:

- no user accounts
- no data collection
- no backend services
- no hidden integrations

All logic is transparent and auditable.

---

## Reporting a Vulnerability

If you discover a security issue related to:

- this repository
- published SafeStack materials
- exposed configuration or unintended data

please report it responsibly.

### Contact

Email:
safestack.tech@pm.me

---

## Responsible Disclosure

- Do not publicly disclose vulnerabilities before review
- Provide clear reproduction steps if possible
- Allow reasonable time for analysis and response

We aim to acknowledge and review reports in good faith.

---

## Out of Scope

The following are not considered vulnerabilities:

- design choices aligned with SafeStack philosophy
- absence of features (e.g., no authentication, no tracking)
- limitations inherent to static content

---

## Philosophy

SafeStack does not rely on trust.

It relies on:
- transparency
- control
- verifiability

> Privacy is not a feature. It is a boundary.
